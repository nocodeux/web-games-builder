# Deploy a TUIFY App via curl (API Render)

The `POST /api/publish/render` endpoint lets you save **and** publish a TUIFY project in a single HTTP call. This is the fastest path from a generated JSON to a live URL.

---

## Authentication

All API calls require a JWT token. Grab yours from the TUIFY editor:  
**Settings Γ¶Ê API tab Γ¶Ê JWT TOKEN Γ¶Ê Copy**

---

## One-call deploy (game world)

```bash
curl -X POST https://tuify.app/api/publish/render \
  -H "Authorization: Bearer <your-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "project": { ...full TUIFY project JSON... },
    "slug": "my-platformer",
    "title": "My Platformer",
    "publishMode": "game"
  }'
```

Response:
```json
{
  "success": true,
  "id": "project-uuid",
  "slug": "my-platformer",
  "url": "https://tuify.app/username/my-platformer",
  "editorUrl": "https://tuify.app/?project=project-uuid"
}
```

Open `url` to play the game. Open `editorUrl` to edit it in the builder.

---

## Request body

| Field | Required | Default | Description |
|-------|----------|---------|-------------|
| `project` | Γ£à | ΓÇË | Full TUIFY project JSON (see `tuify://schemas/project.schema`) |
| `slug` | ΓÇË | derived from `project.name` | URL slug (`a-z0-9-`, 3ΓÇô64 chars) |
| `title` | ΓÇË | `project.name` | Page `<title>` |
| `description` | ΓÇË | ΓÇË | Meta description |
| `publishMode` | ΓÇË | `"game"` | `"game"` ┬╖ `"page"` ┬╖ `"page+game"` |
| `pageHtml` | Γ£à for page | ΓÇË | Pre-rendered HTML string (page/page+game modes only) |

---

## Publish modes

| Mode | What gets published | `pageHtml` needed? |
|------|--------------------|--------------------|
| `game` | Game worlds from the project | No |
| `page` | UI screens as a static HTML page | Yes |
| `page+game` | Both in one page | Yes |

For `page` mode the HTML must be generated by the TUIFY editor's export pipeline. Use `game` mode from the API.

---

## Save only (no publish)

To save a project without publishing, use `POST /api/projects`:

```bash
curl -X POST https://tuify.app/api/projects \
  -H "Authorization: Bearer <your-token>" \
  -H "Content-Type: application/json" \
  -d '{ ...full TUIFY project JSON with id field... }'
```

Then open the editor URL: `https://tuify.app/?project=<project-id>`

## Using the MCP tools

If you have the TUIFY MCP server configured, you can use these tools instead of raw curl:

**`render_app`** ΓÇË save + publish in one call, returns live URL:
```
render_app({
  apiBase: "https://tuify.app/api",
  token: "<your-token>",
  project: { ...project JSON... },
  slug: "my-platformer",
  publishMode: "game"
})
```

**`create_project`** ΓÇË save only, returns editor URL:
```
create_project({
  apiBase: "https://tuify.app/api",
  token: "<your-token>",
  project: { ...project JSON... }
})
```

## Workflow: generate a full app from a prompt

1. Ask Claude (with TUIFY MCP): *"Generate a platformer game with 2 levels, a player, and coin collectibles"*
2. Claude calls `get_project_schema` and `list_components` to understand the format
3. Claude generates the project JSON
4. Claude calls `render_app` with the JSON
5. Claude returns the live URL ΓÇË open it in a browser to play immediately
